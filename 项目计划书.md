# 企业知识库智能体系统项目计划书

## 一、项目定位

项目名称：`enterprise-ai-agent-system`

项目目标：构建一个可用于求职展示的完整智能体项目，覆盖 `RAG`、`ReAct`、`LangGraph`、`FastAPI`、`Vue`、`Docker` 等核心能力，重点体现“能把智能体做成可交付系统”的工程能力。

系统定位：

- 面向企业内部知识库的智能问答与任务处理系统
- 支持企业文档上传、解析、索引与检索
- 支持基于知识库的 RAG 问答
- 支持多轮对话、引用返回、会话记录
- 支持 ReAct 工具调用与 LangGraph 工作流
- 支持服务化部署与项目演示

该项目不是纯演示，而是求职项目。核心原则是：

- 先做通，再做强
- 先保完整，再追求高级
- 先交付一个能演示的版本，再逐步增加工程亮点

交付结果：

- 可以上传企业文档

### 阶段 3：RAG 问答模块

目标：先做出可用的企业问答系统。

要做的事：

- 检索接口
- 问答接口
- 检索结果拼接上下文
- 返回答案和引用来源
- 多轮对话存储
- 流式输出接口

交付结果：

- 用户可在页面提问

### 阶段 4：智能体模块

目标：让系统从“问答”升级为“智能体”。

要做的事：

- 设计 LangGraph 状态

- 增加检索节点
- 增加 ReAct 推理与工具调用节点

- 增加低置信度人工确认节点

交付结果：

- 简单问题走普通 RAG
- 复杂问题走智能体工作流
- 能体现对 ReAct 和 LangGraph 的掌握

- 基础评测数据集

- `.env.example`
- `README`

## 七、智能体工作流设计

建议用 5 个节点：

1. `classifier`
   判断问题类型：普通问答、总结、对比、复杂任务

2. `retrieve`
   从知识库检索相关文本分块

3. `reason_act`
   按 ReAct 思路决定是否继续检索、是否调用工具、是否补充分析

4. `human_review`
   低置信度或高风险操作时人工确认

5. `respond`
   整理最终回答，附引用来源和执行摘要
