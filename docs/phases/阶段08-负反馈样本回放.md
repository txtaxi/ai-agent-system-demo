# 阶段 08：负反馈样本回放

## 本阶段目标

在已有反馈记录和统计面板的基础上，补一层“失败案例回放”能力，让系统不仅能看到总量，还能直接定位哪些回答被用户判定为没帮助，并回到对应运行记录查看执行过程。

## 为什么要做

只有统计数字还不够。

- 统计面板告诉我们“哪里容易出问题”
- 负反馈样本回放告诉我们“具体是哪一次出了问题”

这对 Agent 系统很重要，因为回答质量问题往往不是单点故障，而是由多步执行共同造成的。只有把负反馈样本和运行记录连起来，才能真正进入失败案例分析阶段。

## 本阶段完成内容

### 1. 后端新增负反馈样本查询接口

新增接口：

- `GET /api/v1/feedback/negative-samples`

返回内容包含：

- `feedback_id`
- `agent_run_id`
- `conversation_id`
- `rating`
- `comment`
- `created_at`
- `user_query`
- `route`

### 2. 前端反馈统计页增加负反馈样本区块

在反馈统计页新增：

- 负反馈样本列表
- 展示用户问题、路由、备注、反馈时间
- 提供“查看对应运行记录”入口

### 3. 形成失败案例分析闭环

现在系统已经形成：

```text
用户提问
  ->
Agent 工作流运行
  ->
生成 agent_run
  ->
用户反馈
  ->
反馈统计
  ->
负反馈样本回放
  ->
查看对应运行记录
```

- 系统从“只看反馈统计”升级到“可以回放失败案例”
- 为后续评测优化、问题定位、调试复盘提供更强抓手
- 让项目更接近真实企业 Agent 系统中的质量分析流程
