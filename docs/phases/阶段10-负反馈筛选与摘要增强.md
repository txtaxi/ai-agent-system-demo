# 阶段 10：负反馈筛选与摘要增强

## 本阶段目标

在已有“负反馈样本回放”的基础上，再往前推进一层，让失败案例分析不只是“能点开”，而是“能快速筛选和归类”。

本阶段目标：

- 在负反馈样本中直接展示关键执行摘要
- 支持按路由、工具、知识库、人工确认状态筛选负反馈样本
- 提高失败案例定位效率

## 为什么要做

只有“负反馈样本列表 + 跳转运行记录”还不够高效。

当负反馈变多之后，最先遇到的问题不是“有没有样本”，而是：

- 哪类失败最常见
- 是不是某个工具经常出问题
- 是不是某个知识库经常出问题
- 负反馈是否多发生在需要人工确认的场景

因此需要把样本本身做得更可筛选、更可读。

## 本阶段完成内容

### 1. 负反馈样本补充摘要字段

负反馈样本接口新增返回：

- `selected_tool`
- `retrieval_quality`
- `human_review_required`
- `knowledge_base_names`

这些字段都来自 `agent_runs.trace_json.summary`，不需要再额外查一次运行详情。

### 2. 前端新增筛选能力

反馈统计页的负反馈区块新增筛选项：

- 按路由筛选
- 按工具筛选
- 按知识库筛选
- 按人工确认建议筛选

### 3. 负反馈样本可直接阅读

现在样本卡片里会直接展示：

- 用户问题
- 路由
- 工具
- 检索质量
- 是否建议人工确认
- 知识库名称
- 反馈时间
- 备注

这样只有在真正需要时才跳到运行记录页。
