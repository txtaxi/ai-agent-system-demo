# 阶段 2：知识库实现记录

## 本阶段解决了什么

本阶段解决的是“系统如何拥有知识输入”的问题。

在这一阶段之前，系统只有骨架，没有真正可供检索的数据。

本阶段之后，系统已经具备：

- 知识库容器
- 文档上传
- 文档处理
- 分块结果
- 向量存储

## 关键实现点

### 1. 文档处理状态流转

文档当前支持以下状态：

- `uploaded`
- `processing`
- `processed`
- `failed`

### 2. 多格式解析

已经支持：

- `txt`
- `md`
- `pdf`
- `docx`

### 3. 可插拔分块结构

分块逻辑没有写死在单个函数里，而是抽成了多个分块策略和统一路由入口。

这为后续继续扩展更复杂的结构化分块打下了基础。

### 4. 向量化落地

分块完成后会同步生成向量，并写入数据库。

这让系统正式具备进入检索阶段的基础。

## 本阶段的意义

这一阶段最大的价值在于：

`RAG 的输入层已经成型`

没有这一层，后续的问答、自动选库、任务路由和 Agent 工作流都没有真实数据基础。
