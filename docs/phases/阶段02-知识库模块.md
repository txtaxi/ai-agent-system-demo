# 阶段 2：知识库模块

## 目标

完成知识库、文档、分块和处理链路，建立 RAG 的输入层。

## 设计重点

这一阶段的核心任务不是“回答问题”，而是先把文档处理链路打通。

关键链路如下：

```text
创建知识库
  ->
上传文档
  ->
解析文档
  ->
切分文本
  ->
写入分块
  ->
生成向量
```

## 实际完成内容

### 数据模型

- `knowledge_bases`
- `documents`
- `document_chunks`

### 接口

- `POST /api/v1/knowledge-bases`
- `GET /api/v1/knowledge-bases`
- `POST /api/v1/documents/upload`
- `GET /api/v1/documents`
- `POST /api/v1/documents/{document_id}/process`
- `GET /api/v1/documents/{document_id}/chunks`

### 文档处理能力

- 支持 `txt`
- 支持 `md`
- 支持 `pdf`
- 支持 `docx`

### 分块能力

已从单一规则演进为多策略分块：

- `policy_structure`
- `section_heading`
- `paragraph_first`

### 向量能力

- 分块向量生成
- `pgvector` 存储

## 阶段结论

阶段 2 完成后，系统已经不再是空壳，而是具备了真正的知识入库能力。

这也是后续 RAG、自动选库和 Agent 工作流的前提。
