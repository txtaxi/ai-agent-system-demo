# 阶段 3：RAG 问答与路由

## 目标

在知识库链路已经打通的基础上，完成问答、会话、自动选库、跨库检索和任务路由。

## 本阶段主要完成内容

### 问答能力

- 基础问答接口
- 流式问答接口
- 引用来源返回

### 会话能力

- `conversations`
- `messages`
- 历史会话列表
- 会话消息查看

### 自动选库与跨库检索

- 用户可直接提问
- 系统自动选择相关知识库候选
- 支持跨知识库统一检索
- 前端展示命中的知识库候选

### 任务路由

当前支持三类任务：

- `qa`
- `summary`
- `comparison`

这些类型不再只是换提示词，而是开始走不同的上下文组织流程。

## 当前阶段的系统能力变化

经过阶段 3，系统从：

`知识库处理系统`

升级为：

`企业知识库 RAG 系统`

也就是说，系统不再只是能把文档存进去，而是已经能真正拿这些文档来回答问题。

## 本阶段结论

阶段 3 结束后，系统已经具备：

- 可提问
- 可返回引用
- 可保存会话
- 可自动选库
- 可跨库检索

这使它具备了继续向 Agent 系统演进的基础。
