# 验收清单

## 当前阶段验收目标

当前阶段的验收重点不是“系统是否完美”，而是确认：

- 主链路是否稳定
- 当前阶段功能是否足以继续进入下一阶段
- 关键能力是否已经可以演示

## 一、知识库模块

- 可以创建知识库
- 可以获取知识库列表
- 可以上传文档
- 可以查看文档列表
- 可以处理文档
- 文档状态能够正常变化

## 二、文档处理模块

- `txt` 文件可处理
- `md` 文件可处理
- `pdf` 文件可处理
- `docx` 文件可处理
- 可以查看分块结果
- 分块结果可以写入数据库

## 三、检索与问答模块

- 可以基于知识库问答
- 问答响应包含引用来源
- 自动选库生效
- 跨库检索生效
- 流式问答可用

## 四、会话模块

- 新问题会生成会话
- 会话列表可查看
- 历史消息可查看
- 可以切换不同会话

## 五、任务路由模块

- `qa` 可以命中
- `summary` 可以命中
- `comparison` 可以命中
- 前端能看到路由结果

## 六、Agent 工作流与运行记录

- 新问答会生成一条运行记录
- 运行记录列表可查看
- 可按会话筛选运行记录
- 运行详情可查看
- 详情中可以看到工作流步骤

## 七、容器与部署

- `docker compose up --build` 可以启动
- 前端可访问
- 后端可访问
- 健康检查可通过

## 当前阶段通过标准

如果以上各项大部分都已通过，就说明当前项目已经具备：

- 可演示
- 可继续开发
- 可用于求职讲解

这就足以进入下一阶段，而不需要等所有细节都做完。
