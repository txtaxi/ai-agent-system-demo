# 数据库设计

## 设计目标

当前数据库设计围绕企业知识库问答系统展开，既要支撑基础 RAG 链路，也要为后续 Agent 工作流、运行记录和评测能力预留扩展空间。

当前主要承担以下职责：

- 存储知识库、文档、分块、会话和消息
- 存储向量检索所需的分块向量
- 存储 Agent 工作流运行记录与执行轨迹

## 表结构概览

当前核心表包括：

- `knowledge_bases`
- `documents`
- `document_chunks`
- `conversations`
- `messages`
- `agent_runs`

## knowledge_bases

用途：

- 表示一个知识库实体
- 作为文档归属与检索边界的上层容器

核心字段：

- `id`
- `name`
- `description`
- `created_at`
- `updated_at`

关系：

- 一个知识库下可以有多个文档

## documents

用途：

- 存储上传文档的元数据和处理状态

核心字段：

- `id`
- `knowledge_base_id`
- `filename`
- `file_path`
- `file_type`
- `status`
- `error_message`
- `created_at`
- `updated_at`

状态说明：

- `uploaded`
- `processing`
- `processed`
- `failed`

关系：

- 一个文档属于一个知识库
- 一个文档可以拆分出多个分块

## document_chunks

用途：

- 存储文档切分后的文本分块
- 存储分块向量与分块元数据

核心字段：

- `id`
- `document_id`
- `chunk_index`
- `content`
- `metadata_json`
- `embedding`
- `created_at`

设计说明：

- `metadata_json` 用于记录分块策略、标题、段落信息等附加上下文
- `embedding` 使用 `pgvector` 存储，供相似度检索使用

关系：

- 一个分块属于一个文档
- 通过文档可进一步关联到所属知识库

## conversations

用途：

- 存储对话会话
- 作为消息聚合容器

核心字段：

- `id`
- `title`
- `created_at`
- `updated_at`

关系：

- 一个会话下可以有多条消息
- 一个会话下可以有多次 Agent 运行记录

## messages

用途：

- 存储会话中的用户消息与系统回复

核心字段：

- `id`
- `conversation_id`
- `role`
- `content`
- `created_at`

说明：

- `role` 当前主要包括 `user` 和 `assistant`
- 用于前端历史会话展示和上下文追踪

## agent_runs

用途：

- 存储一次 Agent 或工作流执行记录
- 为后续运行追踪、调试、可观测性和评测提供基础

核心字段：

- `id`
- `conversation_id`
- `user_query`
- `route`
- `status`
- `trace_json`
- `started_at`
- `finished_at`

字段说明：

- `route` 表示本次问题最终走的任务类型，例如 `qa`、`summary`、`comparison`
- `status` 表示运行状态，当前包括 `running`、`completed`、`failed`
- `trace_json` 用于记录 LangGraph 各节点执行轨迹、命中的知识库、检索结果数量和回答摘要

关系：

- 一次运行可以关联到一个会话
- 一个会话可以包含多次运行

## 当前关系结构

```text
knowledge_bases
  -> documents
       -> document_chunks

conversations
  -> messages
  -> agent_runs
```

## 当前设计特点

- 先保证基础 RAG 链路完整可用
- 保留知识库、文档、分块三层关系，便于后续做权限与隔离
- 通过 `agent_runs.trace_json` 保存工作流轨迹，降低前期单独建多张运行明细表的复杂度

## 后续可扩展方向

- 增加 `users` 表，支持登录和权限控制
- 为 `agent_runs` 增加更细粒度的节点耗时与工具调用统计
- 将评测、反馈、人工审核拆成独立表
- 为知识库增加租户、部门或标签隔离字段
