# 接口规范

## 接口风格

- 基础前缀：`/api/v1`
- 默认返回 `JSON`
- 文件上传使用 `multipart/form-data`
- 对话流式接口使用 `text/event-stream`

## 当前接口

### `GET /api/v1/health`

- 健康检查接口

### `POST /api/v1/knowledge-bases`

- 创建知识库

### `GET /api/v1/knowledge-bases`

- 获取知识库列表

### `POST /api/v1/documents/upload`

- 上传文档并关联到知识库
- 查询参数：`knowledge_base_id`

### `GET /api/v1/documents`

- 获取文档列表
- 可选查询参数：`knowledge_base_id`

### `POST /api/v1/documents/{document_id}/process`

- 触发文档解析、切分和向量化

### `GET /api/v1/documents/{document_id}/chunks`

- 获取指定文档的分块结果

### `POST /api/v1/retrieval/search`

- 执行检索调试
- 支持 `knowledge_base_id`
- 支持 `knowledge_base_ids`

### `POST /api/v1/chat/ask`

- 执行普通问答
- 支持自动选库和跨库检索

关键响应字段：

- `conversation_id`
- `agent_run_id`
- `route_type`
- `route_reason`
- `selected_tool`
- `selected_tool_reason`
- `selected_knowledge_bases`
- `citations`

### `POST /api/v1/chat/ask/stream`

- 执行流式问答

`meta` 与 `done` 事件会返回：

- `conversation_id`
- `agent_run_id`
- `route_type`
- `route_reason`
- `selected_tool`
- `selected_tool_reason`
- `selected_knowledge_bases`

### `GET /api/v1/conversations`

- 获取会话列表

### `GET /api/v1/conversations/{conversation_id}/messages`

- 获取指定会话的历史消息

### `GET /api/v1/agent-runs`

- 获取工作流运行记录列表
- 支持按 `conversation_id` 过滤

### `GET /api/v1/agent-runs/{agent_run_id}`

- 获取指定运行记录详情
- 返回 LangGraph 工作流步骤轨迹

## 当前接口关系

```text
知识库接口
  ->
文档接口
  ->
检索接口
  ->
问答接口
  ->
会话接口
  ->
运行记录接口
```
