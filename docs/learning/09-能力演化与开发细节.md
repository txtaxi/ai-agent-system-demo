# 能力演化与开发细节

## 这份文档的目的

这份文档不是面向最终用户的使用说明，而是面向开发者和学习者的“项目能力演化记录”。

它主要回答两个问题：

1. 这个系统是怎么一步一步长到现在的。
2. 每一步是通过哪些文件和哪些改动实现的。

## 一、整体演化链

从项目开始到现在，系统大致经历了下面这条演化链：

```text
项目骨架
  ->
知识库管理
  ->
文档处理
  ->
分块与向量化
  ->
RAG 问答
  ->
会话与流式输出
  ->
任务路由
  ->
自动选库与跨库检索
  ->
Agent Tools
  ->
LangGraph 工作流
  ->
运行记录
  ->
工具选择节点
  ->
大模型辅助工具选择
```

这条链条不是随意堆功能，而是很典型的 Agent 工程推进顺序：

- 先搭底座
- 再打通知识输入
- 再打通检索和问答主链路
- 再引入路由、工具和工作流
- 最后补可观测性与决策能力

`企业知识库 RAG 系统 + 自动选库 + 任务路由 + Agent Tools + LangGraph + 运行记录 + 模型辅助工具选择`

## 十三、下一步最合理的方向

如果继续演化，下一步更合理的是：

- 扩展更多工具类型
- 让模型选择更多工作流分支
- 继续增强运行记录中的工具调用细节
- 再做检索质量优化和评测
